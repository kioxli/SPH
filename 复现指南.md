## 1. 环境要求

### 硬件要求

| 组件 | 最低要求 | 推荐 |
|------|---------|------|
| CPU | 任意现代 x86-64 | — |
| GPU | NVIDIA GPU，算力 ≥ 6.0（Pascal 以上）| 算力 ≥ 7.0，显存 ≥ 6 GB |
| 内存 | 4 GB | 8 GB+ |
| 显存 | 2 GB | 6 GB+ |

**查看 GPU 算力**：在命令行运行 `nvidia-smi`，或在 [NVIDIA 官网](https://developer.nvidia.com/cuda-gpus) 查询型号对应算力。

常见支持型号：GTX 1060/1070/1080、RTX 2060/3060/4060 及以上均可正常运行。

### 软件要求

| 软件 | 版本要求 | 说明 |
|------|---------|------|
| Windows / Linux / macOS | — | 三个平台均支持 |
| Python | 3.8 – 3.12 | Taichi 官方支持范围 |
| CUDA Toolkit | ≥ 11.7 | 由 Taichi 内部管理，无需手动安装 |
| NVIDIA 驱动 | ≥ 525.60（Linux），≥ 528.33（Windows）| 需手动安装 |

## 2. 安装 Python 环境

推荐使用 **Conda** 管理独立的 Python 环境，避免与系统 Python 冲突。

### 方案 A：使用 Conda（推荐）

**步骤 1**：安装 Miniconda（如果尚未安装）

前往 https://docs.conda.io/en/latest/miniconda.html 下载对应系统的安装包并安装。

**步骤 2**：创建专用虚拟环境

```bash
conda create -n fsi python=3.11 -y
```

**步骤 3**：激活环境

```bash
conda activate fsi
```

激活后，命令行提示符前应出现 `(fsi)` 标记。

---

### 方案 B：使用 venv（Python 自带）

**步骤 1**：进入项目目录

```bash
cd C:/Users/kioxli/Desktop/learn/SPH/fsi
```

**步骤 2**：创建虚拟环境

```bash
python -m venv .venv
```

**步骤 3**：激活虚拟环境

Windows（PowerShell）：

## 6. 交互操作说明

运行 `main.py` 时，窗口支持以下交互：

| 操作 | 效果 |
|------|------|
| **鼠标左键点击** | 在鼠标位置添加一个新方块（最多 10 个）|
| **关闭窗口** | 退出模拟 |

**添加刚体的技巧**：
- 点击流体区域上方 → 方块落入水中，被冲走后浮起
- 点击已有水面上 → 方块直接落入流体
- 方块密度为 500 kg/m³（小于水），会最终浮在水面

## 7. 参数调整实验

以下是几个入门级参数实验，修改 `config.py` 后重启程序即可观察效果。

### 实验 1：让刚体下沉

**目标**：将刚体密度改为大于水的密度，使其沉入底部。

打开 [config.py](config.py)，找到第 50 行：
```python
BODY_DENSITY = 500.0   # 默认：会浮
```
改为：
```python
BODY_DENSITY = 1500.0  # 改后：会沉
```

注意：修改此参数后，`BODY_MASS` 和 `BODY_I` 会自动重新计算。

**预期效果**：方块在水中持续下沉，最终静止在底部。

---

### 实验 2：提高流体分辨率

**目标**：更细腻的流体效果（计算量增加）。

找到第 27 行：
```python
PARTICLE_RADIUS = 0.012  # 默认
```
改为：
```python
PARTICLE_RADIUS = 0.008  # 更细，粒子数约增加 2.25 倍
```

**注意**：粒子数按 1/r² 增加，r=0.008 时粒子数约 7000+，对 GPU 要求更高。

**预期效果**：流体细节更丰富，波浪更细腻；运行速度会降低。

---

### 实验 3：更弹性的碰撞

**目标**：让刚体之间碰撞更有弹性。

```python
RESTITUTION = 0.2   # 默认（偏吸能）
```
改为：
```python
RESTITUTION = 0.8   # 高弹性
FRICTION    = 0.1   # 低摩擦
```

**预期效果**：刚体碰撞后弹开幅度明显增大，类似台球效果。

---

### 实验 4：更粘稠的流体

**目标**：模拟粘度更高的流体（如蜂蜜）。

```python
VISCOSITY = 0.05    # 默认（类水）
```
改为：
```python
VISCOSITY = 0.5     # 高粘度
```

**预期效果**：流体流动明显变慢，波浪更平滑，方块更快减速。

---

### 实验 5：更多刚体

**目标**：同时模拟更多刚体（需更多显存）。

```python
MAX_BODIES = 10     # 默认
```
改为：
```python
MAX_BODIES = 20
```

运行后，可以多次点击屏幕添加更多方块，观察多体碰撞效果。

---

### 实验 6：调整溃坝初始尺寸

**目标**：改变初始流体块大小。

在 [particles.py](particles.py) 第 83-84 行，`generate_fluid` 函数默认参数：
```python
def generate_fluid(x_min=None, x_max=0.7, y_min=None, y_max=1.0):
```

在 [main.py](main.py) 调用处可以手动传参，但最简单的方式是直接修改默认值：
```python
x_max=1.2,   # 更宽的初始流体块
y_max=1.5,   # 更高的初始流体块
```

**预期效果**：流体块更大，溃坝冲击力更强，刚体被冲得更远。

## 8. 常见错误排查

### 错误 1：`RuntimeError: [Taichi] No CUDA device found`

**原因**：GPU 不支持 CUDA，或 NVIDIA 驱动未安装。

**解决方案**：
- 确认安装了最新版 NVIDIA 驱动：`nvidia-smi`
- 如果没有 NVIDIA GPU，可改用 CPU 后端（速度慢约 10-50 倍）：
  打开 [main.py](main.py) 第 19 行：
  ```python
  ti.init(arch=ti.cuda)   # 原始
  ```
  改为：
  ```python
  ti.init(arch=ti.cpu)    # CPU 后端
  ```
  同样修改 [validate_hydrostatic.py](validate_hydrostatic.py) 第 17 行。

---

### 错误 2：`ImportError: No module named 'taichi'`

**原因**：Taichi 未安装，或未激活正确的虚拟环境。

**解决方案**：
```bash
# 确认激活了正确的虚拟环境
conda activate fsi   # 或 source .venv/bin/activate

# 重新安装
pip install taichi>=1.7.0
```

---

### 错误 3：窗口一闪而过或立即报错

**可能原因 A**：Taichi kernel 编译报错，查看完整错误信息。

**可能原因 B**：显存不足。

**解决方案**（显存不足）：
打开 [config.py](config.py)，增大粒子半径以减少粒子数：
```python
PARTICLE_RADIUS = 0.016   # 从 0.012 增大，粒子数减少约 44%
MAX_BODIES = 5            # 减少最大刚体数
```

---

### 错误 4：模拟粒子爆炸（粒子飞出屏幕）

**原因**：时间步 dt 过大，数值不稳定。

**解决方案**：减小 `DT_MAX` 或 `CFL_FACTOR`：
```python
DT_MAX     = 2e-4    # 从 5e-4 减小
CFL_FACTOR = 0.3     # 从 0.4 减小
```

或者增大刚度（相当于提高音速，CFL 自动限制 dt）：
```python
STIFFNESS = 100000   # 从 50000 增大
```

---

### 错误 5：窗口可以打开但非常卡顿

**原因**：GPU 性能不足或粒子数太多。

**解决方案**：
```python
PARTICLE_RADIUS = 0.018  # 增大粒子半径
SUBSTEPS = 3             # 从 6 减少子步数
```

---

### 错误 6：`pip install` 速度很慢

**解决方案（中国大陆）**：使用清华镜像源加速：
```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

---

### 错误 7：`validate_hydrostatic.py` 显示 `[FAIL]`

**原因**：通常是 `STIFFNESS` 太小或 `PARTICLE_RADIUS` 太大，导致压力振荡过大。

**解决方案**：
```python
STIFFNESS = 100000   # 增大刚度
# 或者
PARTICLE_RADIUS = 0.010  # 更小粒子半径提高精度
```

## 9. 扩展建议

以下是进一步探索和修改的方向，供有兴趣深入研究的用户参考。

### 9.1 添加顶部边界

目前只有底部和左右两侧有墙壁。在 [particles.py](particles.py) 的 `generate_walls()` 函数中，仿照底部的写法添加顶部边界粒子即可。

### 9.2 修改刚体形状

当前刚体固定为矩形。修改 [particles.py](particles.py) 中的 `generate_rigid_template()` 函数，可以采样不同形状（如三角形、L 形等）的表面粒子。

同时需要修改 [collision.py](collision.py) 中的 `collide_obb()` 函数以支持非矩形碰撞检测（这是较大的工程量，可先从三角形的 SAT 开始）。

### 9.3 导出数据

在 [main.py](main.py) 的主循环中，可以定期将粒子位置导出为 `.npy` 文件：

```python
if frame % 50 == 0:
    x_np = sim.x.to_numpy()[:int(sim.n_active[None])]
    np.save(f"output/frame_{frame:05d}.npy", x_np)
```

### 9.4 使用 Metal 后端（macOS）

macOS 用户将 `ti.init(arch=ti.cuda)` 改为 `ti.init(arch=ti.metal)` 即可在苹果 M 系芯片或 AMD GPU 上运行。

### 9.5 尝试不同场景

在 [main.py](main.py) 中调用 `generate_fluid()` 时传入不同参数，可以快速切换初始场景：

```python
# 窄高水柱（高压溃坝）
fluid_pos, n_fluid = generate_fluid(x_max=0.3, y_max=1.8)

# 宽浅水面
fluid_pos, n_fluid = generate_fluid(x_max=1.8, y_max=0.4)
```

### 9.6 替换 WCSPH 为 PCISPH/DFSPH

当前流体求解器是显式 WCSPH，存在可压缩性和压力振荡问题。更高精度的方案包括：
- **PCISPH**（预测-修正）：迭代求解密度误差，精度更高
- **DFSPH**（散度自由 SPH）：真正的不可压缩，计算量更大

感兴趣可参考 [SPlisHSPlasH](https://github.com/InteractiveComputerGraphics/SPlisHSPlasH) 等开源框架。
