## 1. 项目概述

本项目实现了一个 **实时 2D 流固耦合（FSI: Fluid-Structure Interaction）模拟器**，场景为经典的溃坝（dam-break）冲击可移动刚体。

**模拟的物理现象**：
- 流体在重力作用下塌陷（溃坝），产生波浪冲击刚体
- 刚体受流体压力驱动，可在水中自由平移和旋转（密度 < 水密度时会浮起）
- 刚体之间发生弹性碰撞（摩擦 + 恢复系数可调）

**技术栈**：

| 层级 | 技术 |
|------|------|
| 流体求解 | WCSPH（弱可压缩 SPH）+ Tait 物态方程 |
| GPU 加速 | Taichi Lang（CUDA 后端）|
| 流固耦合 | 虚拟粒子法（Akinci 2012）|
| 刚体碰撞 | SAT + Sequential Impulse（移植自 box2d-lite）|
| 可视化 | Taichi GGUI（canvas.circles + canvas.lines）|

## 3. 核心算法原理

### 3.1 WCSPH — 弱可压缩光滑粒子流体动力学

SPH 是一种无网格粒子方法，将连续流体离散为有限个粒子，通过**核函数加权插值**计算各物理量。

**核函数（Cubic-Spline）**：

```
W(r, h) = k × f(q),   q = |r| / h

         ⎧ 6q³ - 6q² + 1       （q ≤ 0.5）
f(q) =   ⎨ 2(1 - q)³            （0.5 < q ≤ 1）
         ⎩ 0                     （q > 1）

k = 40 / (7πh²)   （二维归一化系数）
```

其中 `h` 为支撑半径（本项目 h = 4r = 0.048 m），超出支撑半径的粒子贡献为零。

**密度估计**：

```
ρᵢ = Σⱼ mⱼ · W(|xᵢ - xⱼ|, h)

其中 mⱼ = ρ₀ · Vⱼ（粒子质量 = 参考密度 × 初始体积）
```

**Tait 物态方程（EOS）**：

"弱可压缩"的含义是允许密度有小幅偏离，通过 Tait 方程将密度变化映射到压力：

```
p = B × [(ρ/ρ₀)^γ - 1]

其中：
  B = 50000 Pa  （刚度系数）
  γ = 7          （Tait 指数）
  ρ₀ = 1000 kg/m³（参考密度）
```

压力梯度力（SPH 离散形式）：

```
(∇p/ρ)ᵢ = Σⱼ mⱼ · (pᵢ/ρᵢ² + pⱼ/ρⱼ²) · ∇W(xᵢ - xⱼ)
```

**人工粘性（Morris 1997）**：

```
aᵥᵢₛ = Σⱼ 8ν · mⱼ · (ρ₀/ρⱼ) · (vᵢ - vⱼ)·rᵢⱼ / (|rᵢⱼ|² + 0.01h²) · ∇W
```

### 3.2 CFL 自适应时间步

CFL（Courant-Friedrichs-Lewy）条件保证数值稳定性：

```
dt = min(DT_MAX,
         α·h / (c + v_max),      ← 声速限制
         α·√(h / a_max))          ← 力限制

其中 α = 0.4（CFL 安全系数），c ≈ 18.7 m/s（人工声速）
```

本项目的 CFL 计算**完全在 GPU 侧**完成（使用 `ti.atomic_max` 归约），无需 CPU↔GPU 同步。

### 3.3 虚拟粒子法（Akinci 2012）

用于处理**流体与固体边界的相互作用**。

**核心思想**：在墙壁和刚体表面均匀采样一层虚拟粒子，赋予"伪质量" ψ，使这些虚拟粒子能参与 SPH 密度和压力计算，从而产生阻止流体穿透的边界力。

**伪质量计算**（预计算一次）：

```
ψⱼ = ρ₀ / [W(0) + Σ_{k≠j} W(|xⱼ - xₖ|)]

含义：ψ 使得边界粒子的自身密度贡献等于参考密度，
      从而在边界附近产生与流体连续的密度分布。
```

**流体受到的边界压力**（来自墙壁/刚体粒子 j）：

```
a_boundary = -mⱼ · (pᵢ/ρᵢ²) · ∇W

其中 mⱼ = ρ₀ · (ψⱼ/ρ₀) = ψⱼ
```

**刚体受到的流体反作用力**（牛顿第三定律）：

```
F_j = ρ₀² · Vᵢ · Vⱼ · (pᵢ/ρᵢ²) · ∇W
```

### 3.4 刚体动力学（牛顿-欧拉积分）

每个刚体是一个**矩形刚体**，具有 3 个自由度：质心位置 (x, y) 和旋转角度 θ。

**速度积分**（在 `_physics_step` 中完成）：

```
F_total = F_fluid + m·g        （流体力 + 重力）
v += dt · (1/m) · F_total
ω += dt · (1/I) · τ_fluid
v *= 0.998  （速度阻尼，防止无限加速）
ω *= 0.998
```

**位置积分**（在 `_finalize_bodies` 中完成，碰撞求解之后）：

```
pos += dt · v
θ += dt · ω
```

**刚体虚拟粒子坐标更新**（每步开始时，在 `_sync_and_assign` 中）：

```
x_world = pos + R(θ) · x_local
v_world = v_rigid + ω × r   （刚体运动公式）
```

### 3.5 均匀网格邻居搜索

SPH 的瓶颈在于为每个粒子找到支撑域内的邻居粒子。本项目使用**均匀网格**加速搜索。

**流程**：
1. 将计算域划分为 h×h 大小的网格格子（约 42×42）
2. 每个粒子根据位置映射到所属格子
3. 搜索时只需遍历中心格子及其 8 个邻格（共 9 个格子）

**内存优化**：使用 Counting Sort + Prefix Sum 将同一格子的粒子排列在连续内存中，显著提升 GPU 缓存命中率。

### 3.6 碰撞检测与求解（SAT + Sequential Impulse）

**SAT（分离轴定理）**：

用于两个 OBB（方向包围盒）的碰撞检测。

```
原理：如果两个凸多边形不相交，则存在一条"分离轴"，
      使两个形状在该轴上的投影不重叠。

对于两个矩形，只需测试 4 个潜在分离轴：
  - A 的两条坐标轴（face_ax, face_ay）
  - B 的两条坐标轴（face_bx, face_by）

若所有轴上投影均重叠 → 碰撞；
选择重叠量最小的轴作为碰撞法线。
```

接触点通过 **Sutherland-Hodgman 裁剪算法**确定（最多 2 个）。

**Sequential Impulse（SI）**：

用于求解接触约束（防止穿透）和摩擦约束。

```
每次迭代对每个接触点：
1. 计算相对速度 vₙ（法向）
2. 法向冲量 ΔPₙ = mass_n · (-vₙ + bias)
   钳位：Pₙ ≥ 0（不能拉伸）
3. 切向冲量 ΔPₜ = mass_t · (-vₜ)
   钳位：|Pₜ| ≤ μ·Pₙ（Coulomb 摩擦锥）
4. 施加冲量到两个刚体的线速度和角速度
```

**Warm-starting**：将上一帧的累积冲量作为本帧的初始猜测，加快收敛。

### 4.2 particles.py — 粒子初始化

**职责**：纯 Python/NumPy 模块，负责生成初始粒子数据并预计算边界伪质量。不依赖 Taichi。

**主要函数**：

| 函数 | 输入 | 输出 | 说明 |
|------|------|------|------|
| `generate_fluid()` | 可选边界范围 | `(N_f, 2) float32`, 粒子数 | 在矩形区域均匀采样流体粒子 |

**初始流体块**（溃坝）：
- x ∈ [margin, 0.7 m]，y ∈ [margin, 1.0 m]
- margin = dx × (WALL_LAYERS + 0.5)，确保流体不与墙壁粒子重叠

**刚体模板（局部坐标）**：

- 所有刚体共享同一模板，世界坐标 = 刚体中心 + R(θ) × 局部坐标

### 4.3 simulation.py — GPU 物理引擎

**职责**：使用 `@ti.data_oriented` 装饰器封装全部 Taichi field 和 kernel，实现完整的 WCSPH 流固耦合仿真。

#### Taichi Field 布局

```
粒子总数 N_max = N_fluid + N_wall + MAX_BODIES × N_per_body

索引约定：
  [0 : N_fluid)         → 流体粒子  (ptype=0)
  [N_fluid : N_base)    → 墙壁粒子  (ptype=1)
  [N_base : N_active)   → 刚体粒子  (ptype=2)
```

**粒子属性 Fields**：

| Field | 类型 | 维度 | 含义 |
|-------|------|------|------|
| `x` | Vector(2) | N | 世界坐标 |
| `v` | Vector(2) | N | 速度 |
| `acc` | Vector(2) | N | 加速度（仅流体粒子使用）|
| `rho` | float | N | 密度 |
| `prs` | float | N | 压力 |
| `mV` | float | N | 等效体积（流体=初始体积，边界=ψ/ρ₀）|
| `ptype` | int | N | 粒子类型（0/1/2）|
| `bid` | int | N | 所属刚体 ID（-1 表示非刚体）|
| `lpos` | Vector(2) | N | 刚体虚拟粒子的局部坐标 |

**刚体属性 Fields**：

| Field | 类型 | 维度 | 含义 |
|-------|------|------|------|
| `b_pos` | Vector(2) | MAX_BODIES | 质心世界坐标 |
| `b_vel` | Vector(2) | MAX_BODIES | 线速度 |
| `b_ang` | float | MAX_BODIES | 旋转角度（弧度）|
| `b_omg` | float | MAX_BODIES | 角速度 |
| `b_frc` | Vector(2) | MAX_BODIES | 累积力（每步重置）|
| `b_trq` | float | MAX_BODIES | 累积力矩（每步重置）|

#### Kernel 详解

**K1: `_sync_and_assign()`** — 刚体同步 + 网格分配
```
输入：刚体位置/角度 (b_pos, b_ang)
输出：刚体粒子世界坐标 (x, v)；网格计数 (g_count)

1. 遍历刚体粒子 → 用刚体变换更新 x 和 v
2. 清空网格计数
3. 遍历所有粒子 → 计算格子 ID，原子累加 g_count
```

**Prefix Sum（`g_prefix.run()`）**：
- 将格子粒子数（`g_count`）变为格子偏移量
- 供 K2 和后续邻居查询使用

**K2: `_grid_sort()`** — Counting Sort 内存重排
```
输入：g_ids（粒子格子ID）、g_count（前缀和）
输出：所有粒子 field 按格子 ID 重新排列

目的：使相邻格子的粒子在 GPU 内存中连续，
      邻居遍历时缓存命中率大幅提升。
```

**K3: `_physics_step()`** — MEGA 物理步（最核心的 Kernel）

此 Kernel 将 6 个逻辑步骤合并为一次 GPU 启动，**避免中间同步**：

```
Loop 1: 密度 + Tait EOS（仅流体粒子）
  → 邻居搜索，累加 mV[j]·W(r) 得到密度
  → Tait EOS 计算压力

Loop 2: 重置刚体力累加器
  → b_frc, b_trq 清零

Loop 3: 力计算（仅流体粒子）
  → 重力
  → 粘性力（来自流体邻居）
  → 压力力（来自流体 + 边界邻居）
  → 如邻居是刚体粒子：通过 atomic_add 累加力到对应刚体
  → 写入 acc[i]

Loop 4: CFL 自适应 dt（GPU 侧完成）
  → 对流体粒子取 max(|v|), max(|acc|)
  → 计算 dt_v, dt_a，取最小值
  → 写入 dt[None]

Loop 5: 流体平流 + 边界钳位
  → v += dt · acc
  → x += dt · v
  → 若越出计算域：钳回并法向速度×(-0.3)反弹

Loop 6: 刚体速度积分
  → 加重力
  → v += dt · F/m，ω += dt · τ/I
  → 速度阻尼 ×0.998
```

**CPU 碰撞求解：`_resolve_collisions()`**：
```
1. 从 GPU field 读取刚体状态（b_pos, b_vel, b_ang, b_omg）到 NumPy
2. 调用 CollisionSolver.solve() 修改速度
3. 写回修正后的速度到 GPU field
```

**K4: `_finalize_bodies()`** — 刚体位置积分
```
（在碰撞求解之后执行，确保速度已修正）
pos += dt · v
θ += dt · ω
边界钳位（撞到域边界时速度×(-0.3)）
```

### 4.4 collision.py — 刚体碰撞

**职责**：纯 Python/NumPy CPU 侧碰撞检测与约束求解，移植自 box2d-lite。

#### 三层架构

**`Contact` 类**：代表一个接触点的所有信息。

```python
Contact.position    # 世界坐标接触点
Contact.normal      # 碰撞法线（从 body1 指向 body2）
Contact.separation  # 穿透深度（负值 = 实际穿透）
Contact.Pn/Pt/Pnb  # 累积法向/切向冲量（warm-start 用）
Contact.mass_n/t   # 有效质量（法向/切向）
Contact.bias       # 偏置速度（位置修正 + 恢复系数）
Contact.r1/r2      # 接触点相对两刚体中心的向量
Contact.feature    # (4,) 接触特征标记，用于跨帧匹配
```

**`collide_obb()` 函数**：SAT 碰撞检测主函数。

```
输入：两个 OBB 的位置、角度、半宽、半高
输出：list[Contact]（0、1 或 2 个接触点）

算法流程：
  1. 将 B 的顶点投影到 A 的坐标系，以及反过来
  2. 计算 4 个潜在分离轴的间隙（face separation）
  3. 若任一轴有正间隙 → 无碰撞，返回 []
  4. 选择间隙绝对值最小的轴（含 rel_tol/abs_tol 容差防抖）
  5. Sutherland-Hodgman 裁剪确定接触边
  6. 从裁剪结果中提取接触点（separation ≤ 0）
```

**`Arbiter` 类**：管理一对刚体的接触点，实现 warm-starting。

```python
Arbiter.update(new_contacts)
  → 新旧接触点按 feature 匹配
  → 匹配上的接触继承 Pn, Pt, Pnb（上帧冲量）
  → 未匹配的接触从零初始化

Arbiter.pre_step(bodies, i, j, inv_dt)
  → 计算每个接触点的有效质量 mass_n, mass_t
  → 计算 bias（穿透修正 + 恢复系数）
  → 施加 warm-start 冲量

Arbiter.apply_impulse(bodies, i, j)
  → 一次 SI 迭代：
     计算并施加法向冲量（钳位 ≥ 0）
     计算并施加切向冲量（Coulomb 摩擦锥钳位）
```

**`CollisionSolver` 类**：顶层接口。

```python
CollisionSolver.__init__(hw, hh)    # 刚体半宽、半高
CollisionSolver.solve(bodies, dt)
  1. O(n²) 遍历所有刚体对，调用 collide_obb
  2. 更新或创建 Arbiter（自动清理无接触的 Arbiter）
  3. pre_step（计算有效质量、施加 warm-start）
  4. SI_ITERS 次迭代，调用 apply_impulse
  5. bodies 列表就地修改 vel, omg
```

### 4.5 main.py — 入口与渲染

**职责**：薄入口层，负责初始化、事件处理和渲染循环。共约 75 行。

**执行流程**：

```python
ti.init(arch=ti.cuda)       # 1. 初始化 Taichi CUDA 后端

data = generate_all()        # 2. 生成粒子数据（NumPy）
sim = FSISimulation(data)    # 3. 创建模拟器（GPU fields 初始化）

window = ti.ui.Window(...)   # 4. 创建 800×800 窗口（无 vsync）

while window.running:        # 5. 主循环
    # 事件：鼠标左键点击 → 添加刚体
    while window.get_event(ti.ui.PRESS):
        if window.event.key == ti.ui.LMB:
            cx, cy = window.get_cursor_pos()  # 归一化坐标 [0,1]
            sim.add_body(cx * DOMAIN_X, cy * DOMAIN_Y)

    # 物理步进（每帧 SUBSTEPS 个子步）
    for _ in range(SUBSTEPS):
        sim.substep()

    # 可视化
    sim.update_vis()
    canvas.circles(sim.x_vis, radius=0.003, per_vertex_color=sim.c_vis)
    canvas.lines(sim.corners, width=0.004, color=(1.0, 0.9, 0.3))
    window.show()
```

**颜色映射**：
- 蓝色 `(0.2, 0.6, 1.0)` → 流体粒子
- 灰色 `(0.45, 0.45, 0.45)` → 墙壁粒子
- 每个刚体一种固定颜色（内置 10 种色板）

**刚体轮廓线**：刚体的 4 条边作为 8 个端点（4 段线段 × 2 端点）存储在 `corners` field 中。

### 4.6 validate_hydrostatic.py — 验证脚本

**目的**：验证 SPH 压力计算的正确性，与解析解对比。

**测试方案**：

```
1. 生成充满域宽的流体柱（高度 1.0 m，约 7000+ 粒子）
2. 将预置刚体移至顶部（不干扰测试）
3. 模拟 4000 个子步，使流体达到静水平衡
4. 将粒子按高度分为 8 个区间
5. 计算每个区间的平均 SPH 压力，与解析解比较：
   p_theory(y) = ρ₀ · g · (H - y)
6. 平均误差 < 20% 则通过
```

**精度特性（WCSPH 固有限制）**：
- 底层（高压）：误差最小，约 2%
- 自由面附近（低压）：误差最大，约 18%
- 平均误差：约 12-16%

这是 WCSPH 的已知特性，主要来源于自由面附近粒子邻域不完整。

## 5. 完整数据流

### 初始化阶段

```
particles.generate_all()
  ├─ generate_fluid()       → 流体坐标 (N_f, 2)
  ├─ generate_walls()       → 墙壁坐标 (N_w, 2)
  ├─ generate_rigid_template() → 刚体模板局部坐标 (N_r, 2)
  ├─ compute_boundary_psi(wall_pos)   → ψ_wall (N_w,)
  └─ compute_boundary_psi(rigid_local) → ψ_rigid (N_r,)
      ↓
FSISimulation(data)
  ├─ _create_fields()       → 分配 GPU Taichi fields
  ├─ _load_initial_data()   → NumPy → GPU（from_numpy）
  └─ CollisionSolver()      → 创建 CPU 碰撞求解器
```

### 每帧（每个 substep）

```
sim.substep()
  │
  ├─ [GPU] _sync_and_assign()
  │       刚体变换同步虚拟粒子坐标/速度
  │       计算网格分配（g_count）
  │
  ├─ [GPU] g_prefix.run(g_count)
  │       Prefix Sum → 格子偏移量
  │
  ├─ [GPU] _grid_sort()
  │       Counting Sort → 按格子重排所有粒子 fields
  │
  ├─ [GPU] _physics_step()（MEGA Kernel）
  │   ├─ Loop 1: 密度计算 + Tait EOS → rho, prs
  │   ├─ Loop 2: 重置 b_frc, b_trq
  │   ├─ Loop 3: 力计算 → acc, b_frc, b_trq
  │   ├─ Loop 4: CFL → dt[None]
  │   ├─ Loop 5: 流体平流 → v, x
  │   └─ Loop 6: 刚体速度积分 → b_vel, b_omg
  │
  ├─ [CPU] _resolve_collisions()
  │       GPU → NumPy（to_numpy）
  │       SAT + SI 碰撞求解
  │       NumPy → GPU（b_vel, b_omg 写回）
  │
  └─ [GPU] _finalize_bodies()
          刚体位置积分 → b_pos, b_ang
          边界钳位
```

### 渲染帧

```
sim.update_vis()
  → 规范化粒子坐标到 [0,1]：x_vis = x / DOMAIN
  → 映射颜色：c_vis（流体/墙壁/刚体各色）
  → 计算刚体轮廓：corners（世界坐标规范化）

canvas.circles(x_vis, radius=0.003, c_vis)
canvas.lines(corners, width=0.004, color=yellow)
```

## 6. 性能优化设计

### 6.1 MEGA Kernel

**问题**：原始实现每个物理步需要 ~16 次 Kernel 启动，每次启动都有 GPU 同步开销。

**方案**：将密度计算、EOS、力计算、CFL 归约、平流、刚体速度积分合并为**单个 Kernel**（`_physics_step`），内部 6 个顺序 for-loop 无需 GPU 同步。

**效果**：每个 substep 仅需 **4 次 Kernel 启动 + 1 次 Prefix Sum**（原 ~16 次）。

### 6.2 合并 Kernel 1

将"同步刚体粒子"和"网格分配"两个操作合并为 `_sync_and_assign`，减少一次 Kernel 启动。

### 6.3 GPU 侧 CFL

CFL 中的最大速度/加速度归约通过 `ti.atomic_max` 在 GPU 侧完成，**完全避免** CPU↔GPU 数据传输。

### 6.4 Counting Sort

粒子按格子 ID 排序后，邻居遍历时访问的粒子在内存中连续，GPU L1 缓存命中率大幅提升。

### 6.5 碰撞 CPU 侧处理

刚体碰撞（SI 迭代）在 CPU 侧完成，避免在 GPU 上处理刚体间的串行依赖关系。仅在 `nb ≥ 2` 时触发，单刚体场景无额外开销。

## 7. 参数全表与调节建议

以下所有参数均位于 [config.py](config.py)。

### 计算域

| 参数 | 默认值 | 单位 | 说明 |
|------|--------|------|------|
| `DOMAIN_X` | 2.0 | m | 域宽 |
| `DOMAIN_Y` | 2.0 | m | 域高 |

### SPH 离散化

| 参数 | 默认值 | 单位 | 调节说明 |
|------|--------|------|---------|
| `PARTICLE_RADIUS` | 0.012 | m | **最重要参数**。减小 → 更高分辨率，粒子数按 1/r² 增加，计算量倍增 |
| `SUPPORT_RADIUS` | 4r = 0.048 | m | 固定为 4 倍粒子半径，改动需谨慎 |
| `M_V0` | 0.8×(2r)² | m² | 流体粒子初始体积，影响质量密度 |

### 物态方程

| 参数 | 默认值 | 单位 | 调节说明 |
|------|--------|------|---------|
| `DENSITY_0` | 1000.0 | kg/m³ | 参考（水）密度，一般不改 |
| `STIFFNESS` | 50000 | Pa | 越大 → 越不可压缩（更稳定但 dt 更小）|
| `EXPONENT` | 7 | — | Tait 指数，通常取 7（水）|

### 粘性

| 参数 | 默认值 | 单位 | 调节说明 |
|------|--------|------|---------|
| `VISCOSITY` | 0.05 | m²/s | 增大 → 更粘稠；过大会阻尼掉流体运动 |

### 时间步进

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `DT_MAX` | 5e-4 s | CFL 上限，减小 → 更稳定但更慢 |
| `DT_MIN` | 1e-5 s | CFL 下限，防止 dt 过小时卡死 |
| `CFL_FACTOR` | 0.4 | 安全系数，增大 → 更快但可能不稳定 |
| `SUBSTEPS` | 6 | 每帧子步数，增大 → 视觉速度加快 |

### 刚体

| 参数 | 默认值 | 单位 | 调节说明 |
|------|--------|------|---------|
| `BODY_W` | 0.3 | m | 刚体宽度 |
| `BODY_H` | 0.2 | m | 刚体高度 |
| `BODY_DENSITY` | 500 | kg/m³ | < 1000 → 刚体会浮；改为 > 1000 会沉 |
| `MAX_BODIES` | 10 | — | 最多刚体数（受 GPU 内存限制）|

### 碰撞

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `FRICTION` | 0.3 | Coulomb 摩擦系数，越大碰撞越不光滑 |
| `RESTITUTION` | 0.2 | 恢复系数（0=完全非弹性，1=完全弹性）|
| `SI_ITERS` | 10 | SI 迭代次数，增大 → 约束更精确但更慢 |
